# Security Policy

## Supported Versions

At this stage, only the latest state of the default branch is considered supported for security fixes.

## Reporting a Vulnerability

Do not open a public issue with exploit details.
Report security issues privately to maintainers with:

- affected component/module
- impact and risk level
- reproduction steps (if available)
- optional mitigation proposal

## Response Targets

Maintainers aim to:

- acknowledge receipt within 7 days
- triage severity and scope
- coordinate remediation and disclosure

## Security Scope

Security-sensitive areas in this repository include:

- networking and RPC provider integration
- API key handling and storage
- logging and error-reporting paths
- local persistence and backup behavior
